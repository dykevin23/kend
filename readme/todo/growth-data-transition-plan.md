# 성장 데이터 더미 → 실데이터 점진적 전환 계획

> 작성일: 2026-04-22  
> 목적: 백분위 비교용 성장 데이터를 초기 **더미(국가 통계 기반)** → **실사용자 데이터**로 점진 전환하는 전략 정리

---

## 1. 현 상태

### 1.1 데이터 구성

- `children` 테이블에 `is_dummy` boolean 컬럼 존재
- 현재 DB에 삽입된 더미: **아이 100명 + 성장기록 3,674건**
  - 질병관리청 2017 소아청소년 성장도표의 분포(LMS)를 기반으로 생성
  - 동일 성별·월령 표본 충분히 확보 목적

### 1.2 백분위 계산 방식

- **이전**: LMS 공식(질병관리청 기준)으로 계산 → 절대 백분위
- **현재**: Supabase RPC `get_growth_percentile_history`로 실제 DB 전체 비교 → **상대 백분위** (`내 아이보다 낮은 값 / 전체 × 100`)
- **결과**: 초기에는 더미가 표본의 대부분이므로 **사실상 국가 통계 기반과 동일한 분포**를 보이지만, 실사용자가 많아지면 **Kend 사용자 집단 특유의 분포**로 점점 이동

### 1.3 사용 UI

- 자녀 성장기록 페이지 차트 3종 (신장, 체중, 머리둘레)
- 자녀의 백분위 추이 + "100명 중 N등" 순위 표시

---

## 2. 문제 의식

현 구조의 한계:

1. **더미가 영구적이면 안 됨**  
   국가 통계는 기준일 뿐, Kend 사용자 집단의 실제 분포를 대변하지 못함. 계속 섞여 있으면 Kend 특유의 인사이트(지역/연령대별 편차 등) 추출 불가

2. **표본 대표성 문제**  
   월령 구간별로 실사용자 표본이 충분해야 상대 백분위가 의미 있음. 초기에는 표본이 적어 통계적으로 불안정

3. **이상치(Outlier) 처리 부재**  
   실데이터에 극단값이 섞이면 백분위가 왜곡됨. 현재는 필터링 로직 없음

4. **전환 기준 없음**  
   "언제 더미를 얼마나 줄일지" 기준이 없음 → 결정이 미뤄지면 더미가 그대로 남음

---

## 3. 전환 전략 (3단계)

### Phase A: 하이브리드 (현재 ~ 수개월)
- **더미 100%** 사용 (실사용자 표본 부족)
- 실사용자 성장기록은 추가되지만, 백분위 계산 대상에 그대로 포함
- 모든 사용자(더미+실) 기준 상대 백분위 계산
- **특별한 개입 없음**, 다만 모니터링 시작

### Phase B: 비중 조절 (실사용자 표본 충분 시)
- **월령 구간별 실사용자 표본이 최소 기준 이상 확보되면** 해당 구간의 더미 비중을 줄이기 시작
- 기준 예시:
  - 월령별 실사용자 성장기록 **30건 이상** 확보 시 → 해당 구간 더미 50%로 축소
  - 100건 이상 → 더미 20%로 축소
  - 300건 이상 → 해당 구간 더미 완전 제외 (`is_dummy = false` 데이터만 사용)
- RPC 함수에 월령 구간별 분기 로직 추가

### Phase C: 실데이터 전환 완료
- 모든 월령 구간에서 실사용자 표본이 충분 → **더미 완전 제외**
- `is_dummy = true` 데이터는 **보존하되 백분위 계산에서 제외**
- 국가 통계는 "참고 레퍼런스"로만 UI에 표시 가능 (옵션)

---

## 4. 전환 지표 (모니터링)

### 4.1 표본 지표
- 월령 구간별 실사용자 성장기록 수
- 성별×월령 매트릭스 커버리지 (빈 구간 체크)
- 활성 자녀 수 (최근 N개월 이내 기록된)

### 4.2 분포 지표
- 실사용자만의 백분위 분포 vs 더미 포함 백분위 분포 차이 (KL-divergence 등)
- 평균/표준편차 변화 추이
- 국가 통계와의 편차

### 4.3 이상치 지표
- 월령 구간별 평균 ± 3σ 밖 비율
- 동일 자녀의 성장기록 급변 탐지 (데이터 오입력 의심)

---

## 5. 구현 시 필요한 것

### 5.1 DB / 쿼리
- `children.is_dummy` 컬럼: 이미 존재
- RPC `get_growth_percentile_history` 확장:
  - 월령 구간별 실사용자 표본 수에 따라 더미 포함 여부 분기
  - 파라미터: `dummy_weight_threshold` 등 동적 조절 가능하게
- 이상치 필터링 로직 추가 (예: 월령별 ±3σ 범위)

### 5.2 운영 도구
- 월령 구간별 표본 수 조회 쿼리 / 대시보드
- 분포 변화 모니터링 리포트 (주기적)
- 더미 비중 조절 파라미터 (환경변수 또는 DB 설정 테이블)

### 5.3 문서화
- 전환 단계별 기준값 확정 (30건/100건/300건 같은 구체 수치는 설계 후 재조정)
- 운영 체크리스트 (주기적으로 모니터링할 지표 정리)

---

## 6. 단계별 작업 예상

| 단계 | 시점 | 작업 |
|------|------|------|
| Phase A 유지 | 현재 | 모니터링 쿼리/대시보드 구축 |
| Phase A → B 전환 | 실사용자 표본 확보 시 | RPC에 비중 조절 로직 추가, 이상치 필터링 |
| Phase B → C 전환 | 모든 구간 충분 시 | 더미 제외 정책 적용 |

Phase A → B 전환 시점은 **사용자 증가 속도에 따라 달라짐**. 출시 후 수개월~수년 단위로 예상.

---

## 7. 미결정 사항

- [ ] 월령 구간별 "충분한 표본" 기준 수치 확정 (30건/100건/300건은 잠정값)
- [ ] 월령 구간 세분화 기준 (1개월 단위? 3개월? 6개월?)
- [ ] 이상치 필터링 방식 (±3σ vs IQR 기반 vs percentile trimming)
- [ ] 성별 외 다른 요인(지역/가정환경 등) 고려 여부
- [ ] 국가 통계 참고 레퍼런스를 UI에 계속 노출할지
