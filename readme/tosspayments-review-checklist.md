# TossPayments 실키 심사 대비 체크리스트

> **목적**: 법인 설립 직후 TossPayments 실키 신청 시 **반려 없이 한 번에 통과**하는 것이 목표.
> **왜 중요한가**: 심사 기간(2~4주)이 출시 일정의 Critical Path. 반려 시 +1~2주 지연 → 9월 말 출시 전제 붕괴.
> **관련 문서**: [kend-roadmap-to-launch.md](kend-roadmap-to-launch.md), [kend-milestones.md](kend-milestones.md) (해당 시)

---

## 심사 구조

### 1단계: TossPayments 내부 심사 (1~2주)
- 서류 심사 + 사이트/앱 실사 (실제 접속해서 확인)

### 2단계: 카드사별 심사 (병렬 1~2주)
- Toss가 각 카드사(KB/신한/삼성/현대/롯데 등)에 가맹점 등록 요청
- 카드사마다 심사 속도 다름 → 일부 카드는 먼저 결제 가능, 일부는 늦게 열림
- **"일부 카드사 심사 지연"은 정상** — 출시 시점에 주요 카드(삼성/신한/KB) 열려있으면 운영 가능

### 총 소요: 평균 2~4주
- 서류 보완 요청 시 **+1~2주 지연**
- 고위험 업종 분류 시 더 오래 걸림 (KEND는 해당 없음 예상)

## 사이트/앱 사전 점검 체크리스트 ★심사 직전 반드시 확인★

Toss 담당자가 **실제 접속해서 확인**하는 항목들. 하나라도 빠지면 반려 또는 보완 요청.

### 기본 노출 (Footer / 약관 / 마이페이지 어딘가)
- [ ] **상호명** (법인 정식 명칭)
- [ ] **대표자명**
- [ ] **사업장 주소**
- [ ] **연락처 (고객센터 전화번호)** — 휴대폰 X, 대표번호 권장
- [ ] **이메일 (CS용)**
- [ ] **사업자등록번호** (10자리)
- [ ] **통신판매업 신고번호**
- [ ] **개인정보보호책임자** 이름/연락처

### 구매/결제 플로우
- [ ] 상품이 **실제로 조회 가능**한 상태 (플레이스홀더/더미 X)
- [ ] 결제 금액, 배송비, 최종 결제금액이 명확히 표시
- [ ] 배송 정보 입력 화면 작동
- [ ] 결제 수단 선택 화면 노출
- [ ] **취소/환불 요청 경로가 앱/사이트 내에 존재**
- [ ] 주문 완료 후 **주문 내역 조회** 가능

### 약관 및 정책 (접근 가능한 경로에 존재)
- [ ] 이용약관
- [ ] 개인정보 처리방침
- [ ] **환불/반품/교환 정책** — 전자상거래법상 7일 이내 청약철회 규정 포함 필수
- [ ] 결제 관련 약관 (전자금융거래 이용약관)

### 판매자(Seller) 노출 (오픈마켓 구조인 경우)
- [ ] 각 상품 상세에 **판매자 정보 노출** (상호/사업자번호 등)
- [ ] 판매자별 환불/교환 정책 안내
- [ ] "통신판매 중개자이며 개별 판매자의 상품을 중개" 라는 명시

## KEND 특성별 리스크

### 🟥 리스크 1: 오픈마켓(통신판매중개) 분류

KEND는 Seller(판매자)가 상품을 등록·판매하는 구조 → **통신판매중개업** 으로 분류될 가능성 높음.

**영향**:
- 일반 자사몰보다 심사 항목이 **더 엄격**
- 판매자 정보 관리 체계 / 정산 로직까지 심사 범위
- 추가 서류나 질의 올 수 있음

**대응**:
- Seller 사업자 정보 수집/검증 flow 명확화 (Phase 2 `P2-1 판매자 기반`)
- 상품 상세에 판매자 정보 노출 (상호/사업자번호/연락처)
- "당사는 통신판매중개자이며 개별 거래의 당사자가 아님" 문구 삽입 (법적 책임 한정)

### 🟥 리스크 2: 에스크로 의무

**규정**: 전자상거래법상 **10만원 이상 선불결제 시 에스크로 또는 소비자피해보상보험 의무**.

**영향**:
- 고가 상품 판매 시 에스크로 미적용이면 심사 반려 가능
- TossPayments는 에스크로 옵션 제공 — 신청 시 같이 요청 필요

**대응**:
- 판매 상품의 가격대 정책 결정: 고가 상품이 있다면 **Toss 에스크로 옵션 활성화 신청**
- 저가만 취급하면 제외 가능

### 🟡 리스크 3: 아동 관련 서비스 (개인정보)

아동 데이터(성장기록 등)를 취급 → **개인정보 처리방침에 아동 관련 조항 필수**.

**영향**:
- 만 14세 미만 가입 시 **법정대리인 동의 절차** 필요 (정보통신망법)
- 회원가입 flow에 연령 확인 및 보호자 동의 없으면 약관 심사에서 걸림

**대응**:
- 현재 KEND가 **가입 연령 정책**을 어떻게 정하고 있는지 확인 필요
  - 안 (A): 가입은 성인(보호자)만, 자녀는 프로필로 관리 → 가장 깔끔
  - 안 (B): 만 14세 이상 자녀 가입 허용 → 법정대리인 동의 절차 필요
- 개인정보 처리방침에 **자녀 정보(생년월일/신장/체중) 수집 목적 및 보관 기간 명시**

### 🟡 리스크 4: 판매자 정산 체계

오픈마켓이면 **판매자에게 돈이 언제 어떻게 가는지** 를 심사에서 볼 수 있음.

**영향**:
- 정산 주기, 수수료, 에스크로 해제 조건 등이 불투명하면 재질의
- Phase 3 정산 시스템 완성 전 심사면 "정산 예정 로직" 설명 자료 요구될 수 있음

**대응**:
- 판매자 가입 약관에 정산 규정 명시 (주기, 수수료율, 구매확정 후 N일)
- Phase 3 정산 로직 설계 시 심사 대비 문서 1장 준비

## 일정 의존성 (Critical Path)

```
법인 설립 완료 (대표 진행, 법무사)
  ├─→ 통신판매업 신고 (같이 위임, 1~2주)
  │     ↓
  │   법인 사업자등록증 + 통신판매 신고번호 확보
  │     ↓
  ├─→ TossPayments 실키 신청 (★ 법인 완료 당일 바로)
  │     ├─ Toss 내부 심사: 1~2주
  │     └─ 카드사 심사: 1~2주 (병렬)
  │          ↓
  │        실키 발급 ← 여기가 Critical Path
  │
  └─→ NICE 본인확인 계약 (1~2주)
```

**가장 중요한 원칙**:
1. **법인 설립 = TossPayments 신청 트리거** — 하루라도 지체하지 말 것
2. **통신판매업 신고는 법인 설립과 함께** — 순차 진행하면 1~2주 낭비
3. **사이트/앱 점검은 신청 **전** 완료** — 심사 중 추가 수정이면 재심사

## 액션 아이템 (지금부터)

### 즉시 (4월 말 ~ 5월)
- [ ] 대표에게 **"법인 설립 시 통신판매업 신고도 같이 위임"** 요청 전달
- [ ] 대표에게 **법인 사업자 정보 확정 (상호/주소/업종)** 공유 받기
- [ ] CS 연락처/이메일 정책 결정 (대표번호 vs 070 가상번호)
- [ ] 판매 상품 가격대 정책 결정 → 에스크로 필요 여부 판단

### Phase 1 중 (5월~6월)
- [ ] 사이트/앱 footer에 **사업자 정보 노출 영역 설계** (법인 정보 확정되면 바로 투입 가능하도록)
- [ ] 이용약관/개인정보 처리방침 보완 초안 작성
  - [ ] 전자상거래법 청약철회 조항 확인
  - [ ] 아동 개인정보 취급 조항 추가
  - [ ] 통신판매중개자 지위 명시
- [ ] 환불/반품 정책 문서 작성 + 노출 경로 확정

### Phase 2 중 (6월~7월)
- [ ] Seller 사업자 정보 수집 flow 설계 (`P2-1 판매자 기반`)
- [ ] 상품 상세에 판매자 정보 노출 영역 추가
- [ ] 통신판매중개 면책 문구 삽입

### 법인 설립 완료 직후 (6월 초 목표)
